# Pong Quest (SFML)\n\nA simple, menu-driven paddle-and-ball game built in C++ with SFML. \n\n## 🎮 Game Overview\n- Controls: left/right arrow keys to move the paddle.\n- Goal: keep the ball bouncing and avoid letting it fall below the paddle.\n- Score increments each time the ball hits the paddle.\n- Difficulty increases every 5 points (ball speed increases, level up).\n- Tracks highest score across runs in `highest_score.txt`.\n- Menu options: Play, Credits, Exit.\n\n## 🧩 Features\n- `MENU`, `GAME`, `CREDITS`, `GAMEOVER` states.\n- Text display: current score, level, highest score, game over overlay.\n- Background images and textured sprites.\n- Music + game over sound.\n- Persistent high-score storage.\n\n## 📁 Files\n- `Second sfml.cpp` - main source code (game implementation).\n- `highest_score.txt` - stores highest score persistently.\n- `brickwall.png` - paddle texture.\n- `bowling.png` - ball texture.\n- `b99.jpg` - menu background.\n- `b20.png` - game background.\n- `StonePunk-eZD4g.ttf` - font.\n- `music.ogg`, `gameover.ogg` - audio assets.\n\n## 🛠️ Prerequisites\n- Windows OS\n- C++ compiler (e.g., MSVC, g++)\n- SFML 2.x (Graphics, Window, System, Audio)\n\n## 🏗️ Build Instructions\n1. Install SFML and ensure include/lib paths are configured.\n2. Place all asset files in the executable working directory or adjust paths in source.\n3. Build with your IDE or command-line:  
   - Visual Studio: use the provided `2nd SFML.cbp` project or configure `x64/Win32` with SFML link libraries.\n   - g++ example:  
     `g++ "Second sfml.cpp" -o PongQuest -lsfml-graphics -lsfml-window -lsfml-system -lsfml-audio`\n4. Run the executable (`PongQuest.exe`).\n\n## 🚀 Usage\n- Start the game and click `Play` in the menu.\n- Use `Left`/`Right` arrow keys.\n- Reach higher score thresholds to ramp up difficulty.\n- After game over, wait 5 seconds; then it returns to the menu and saves high score.\n\n## 🎯 Improvement Ideas\n- Add brick blocks and multiple ball lives.\n- Add pause/resume and difficulty settings.\n- Add power-ups (score multiplier, slow ball, bigger paddle).\n- Add sound on paddle hit, wall bounce, and point milestones.\n\n## 📌 Notes\n- If assets fail to load, fallback is logged to console.\n- `highest_score.txt` will be created if missing.\n- The code is meant to be beginner-friendly and extendable.\n\n---\nMaintained by Rik (ETE, RUET, Roll: 2204051)
